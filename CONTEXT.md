# CONTEXT — the efficacy-study domain language

Glossary for the Soul System's value-evaluation work (the "is it real or placebo"
question). Terms resolved in grilling sessions with the Body; keep entries
domain-meaningful, not implementation-coupled.

## The Efficacy Question
The standing fundamental question (Body, 2026-06-10): *does running real projects
under the Soul System actually improve outcomes and the user-AI working relationship,
or does it only feel like it does?* The Body already uses the Soul on most projects —
this is a keep/drop/slim decision about a live practice, not a prospective adoption
decision. Evidence must be tests/benchmarks, not impressions.

## Comparator
What Soul-on must beat to count as real: a **well-written, project-specific,
Soul-free CLAUDE.md of similar length and quality** (the v1.0 study's "arm B" shape).
Decided 2026-06-10. NOT bare Claude (that bar only proves "context helps" — it is a
floor arm at most). Rationale: the Body would write good instructions regardless;
only the Soul-specific machinery's marginal value answers the Efficacy Question.

## Decision-shaped value (from the study record)
The v1.0 study's characterization of where measured Soul value concentrates: it
protects *decisions* (counter-default conventions, unguessable facts) across time and
pressure; it does not make any single session smarter or code generically better.
Measured: record-carriage 5/5 vs 0/5 (F044); force-at-site holds where propositions
mutate (F053); false-completion 6/6 → 0/12 under v1.1 (A019). Counter-evidence at
short horizon: +4.5–20% cost premium; record-accumulation null at 8-task scale
(stream phase, DO-NOT-BUILD); good generic doctrine ties several cells.

## Primary endpoints (the Efficacy Question's decisive metrics)
Decided 2026-06-10. A Soul-vs-Comparator difference here changes the Body's keep/drop/
slim decision; everything else is secondary/exploratory.
- **Body-intervention rate** — corrections, reminders, re-explanations per task (the
  Ad Hoc Methodology rate). The user-AI relationship metric.
- **False-"done" rate** — precision of completion claims (when the AI says done, is
  it). Already shows a measured v1.1 effect (6/6 → 0/12 in vivo).
- **Redo cycles** — tasks reopened after the Body considered them settled; plus
  Body-turns-to-accepted-result.
Secondary: decision-survival, post-acceptance defects, code cleanliness.
Economics (cost per accepted deliverable, wall-clock) decisive only if primaries null.

## The blind twin-arm trial (the Efficacy Question's vehicle)
Decided 2026-06-10: sampled REAL upcoming tasks each run twice in isolated worktrees
(Soul vs Comparator), multi-session tasks included (≥3 sessions — where
decision-shaped value claims to live); the Body reviews outputs blind and scores.
Retrospective transcript mining may supply context, never the verdict.
- **Sampling:** pre-registered eligibility, then the NEXT 12 eligible real tasks as
  they arise across 2–3 active projects — no picking. ≥4 multi-session.
- **Blinding (rubric-first + normalize):** task rubric written before outputs are
  seen; a sanitizer strips brand vocabulary (rule/finding citations, gate lines) but
  keeps substantive reasoning; the Body scores against the rubric BEFORE guessing the
  arm; the guess is recorded to measure blinding leakage. Partial blinding is named,
  not pretended away.
- **Comparator authorship:** a Soul-free session (no doctrine in context) writes the
  best project-specific CLAUDE.md it can from field best practice; the Body signs off
  with the test "would I genuinely run this project on this file if the Soul didn't
  exist?"
- **Power honesty:** n=12 gives decision-grade evidence, not publication-grade proof
  — the preference margin alone cannot reach significance at this n; the
  margin+mechanism rule deliberately leans on the mechanism metrics as co-evidence.

## Decision rule (pre-registered, margin + mechanism)
- **KEEP**: blind preference ≥60% Soul AND a measurable redo/false-done advantage.
- **SLIM** to the measured core (record-force at site, anchor discipline, blocking
  completion gate): preference ~50/50 but drift-class wins persist.
- **DROP**: Comparator wins preference, or all primary endpoints null at n≥10 tasks.
Nulls are honored (precedent: the stream phase's DO-NOT-BUILD).

## Placebo hypothesis
The live alternative the Efficacy Question must rule out: the Soul's apparent value
is ceremony plus the generic benefit of having any good doctrine, and the
drift-class failures it provably prevents are too rare in the Body's real projects
to justify the premium.
