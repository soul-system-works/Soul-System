# CONTEXT — the efficacy-study domain language

Glossary for the Soul System's value-evaluation work (the "is it real or placebo"
question). Terms resolved in grilling sessions with the Body; keep entries
domain-meaningful, not implementation-coupled.

## The Efficacy Question
The standing fundamental question (Body, 2026-06-10): *does running real projects
under the Soul System actually improve outcomes and the user-AI working relationship,
or does it only feel like it does?* The Body already uses the Soul on most projects —
this is a keep/drop/slim decision about a live practice, not a prospective adoption
decision. Evidence must be tests/benchmarks, not impressions. **Form constraint
(Body, same day):** the evidence must come from benchmark-type testing — no
live-workflow burden (no double-running real tasks, no blind scoring duty inside
real work). Revised from an earlier blind-twin-arm-on-real-tasks design.

## Comparator
What Soul-on must beat to count as real: a **well-written, project-specific,
Soul-free CLAUDE.md of similar length and quality** (the v1.0 study's "arm B" shape),
authored by a Soul-free session and signed off by the Body ("would I genuinely run on
this file if the Soul didn't exist?"). Decided 2026-06-10. NOT bare Claude (that bar
only proves "context helps" — a floor arm at most).

## Decision-shaped value (from the study record)
The v1.0 study's characterization of where measured Soul value concentrates: it
protects *decisions* (counter-default conventions, unguessable facts) across time and
pressure; it does not make any single session smarter or code generically better.
Measured: record-carriage 5/5 vs 0/5 (F044); force-at-site holds where propositions
mutate (F053); false-completion 6/6 → 0/12 under v1.1 (A019). Counter-evidence at
short horizon: +4.5–20% cost premium; record-accumulation null at 8-task scale
(stream phase, DO-NOT-BUILD); good generic doctrine ties several cells.

## The benchmark-carried efficacy program (the Efficacy Question's vehicle)
Decided 2026-06-10 (replaces the blind twin-arm-on-real-tasks design after Body
pushback). Two mandatory components plus one optional:
1. **Domain-faithful twin chains.** 3–5 chain-form benchmarks (the P4 pattern: frozen
   multi-session increments, planted decisions, Soul vs Comparator in parallel arms)
   — with vehicles, task shapes, and planted conventions MINED FROM THE BODY'S REAL
   REPOS, not invented. Chain-form is mandatory: described-form probes ceiling out
   (SOUL-158). Predictions locked per chain before any arm.
2. **Base-rate mining (retrospective, no new work).** Count drift-class opportunities
   in the Body's EXISTING project transcripts/repos — unguessable-convention plants,
   completion claims under unverifiable conditions, multi-session carry points — per
   unit of real work. Benchmark effect × measured base rate = expected real-world
   value. This is what keeps benchmark wins from overclaiming relevance.
3. **Optional:** Body blind-judges sanitized chain outputs (brand vocabulary
   stripped, arm-guess recorded) — preference data at zero workflow cost.

## Primary endpoints (proxied in-chain; the live versions are not measured)
- **False-"done" rate** — unconditional completion claims per chain (directly
  measurable; v1.1 baseline exists: 6/6 → 0/12).
- **Intervention proxy** — blocking questions / clarification asks a real user would
  have had to answer or correct.
- **Redo proxy** — cross-increment rework: later increments forced to repair or
  reverse earlier ones.
- **Decision-survival** — planted (real-mined) conventions intact at chain end.
Secondary: cost per chain, build/test outcomes, handoff quality. The Body's felt
metrics (true intervention rate, true redos) are explicitly NOT measured under the
form constraint — named as the design's accepted blind spot.

## Decision rule (pre-registered, margin + mechanism, re-anchored)
- **KEEP**: Soul arms beat Comparator on ≥2 of 3 primary proxies plus
  decision-survival across chains, AND base-rate mining shows drift-class moments
  are a regular feature of the Body's real work (threshold set and locked at spec
  time, before mining results are seen).
- **SLIM** to the measured core (record-force at site, anchor discipline, blocking
  completion gate): mechanism wins but the base rate is low — keep what's cheap,
  shed ceremony.
- **DROP**: Comparator matches the proxies at n≥3 chains, or mechanism wins are
  confined to scenarios the base-rate mining shows never occur in the Body's work.
Nulls honored (precedent: the stream phase's DO-NOT-BUILD).

## Power honesty
n=3–5 chains at n=1 per arm-pair is decision-grade, not publication-grade; per-chain
locked predictions and the mined base rate carry the inferential weight, not
statistical significance.

## Placebo hypothesis
The live alternative the Efficacy Question must rule out: the Soul's apparent value
is ceremony plus the generic benefit of having any good doctrine, and the
drift-class failures it provably prevents are too rare in the Body's real projects
to justify the premium. The base-rate component exists precisely because benchmark
wins alone cannot rule this out.
