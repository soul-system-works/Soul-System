# Soul System

A living philosophy and operational framework for human-AI collaborative work
in research, science, engineering, and coding.

---

## What This Is

Most AI-assisted work fails not because the AI lacks capability
but because the problem was never understood deeply enough before work began.
The AI amplifies whatever framing it receives — including incomplete framing.

This system is the response. A philosophy, a set of operational documents,
and a growing collection of skills and hooks that together produce a more
principled, self-correcting approach to AI-assisted work.

It is not a prompt collection. It is not a workflow template.
It is closer to a craft tradition — principles forged from real failure patterns
that practitioners have recognized as true.

## How to Start

**1. Read the Soul.** `philosophy/the-soul.md` — not as instructions; as a philosophy that must be understood before it is enacted. The one document that rewards slow reading.

**2. Copy `operations/` into your project.** Record the system version from `SYSTEM-VERSION.md`.

**3. Fill in the problem slot.** Open `autonomous-session-template.md`, insert your problem, set a check-in cadence, start the session.

**4. Read the Witness log.** Not the solution — the log. The quality of the log tells you more about what happened than the output does.

**5. Contribute what you find.** See `CONTRIBUTING.md`.

---

## What This Is Not

This is not a guarantee of better output.
It is a guarantee of more honest process.

A rich Witness log from a messy run is a successful exercise.
A smooth run with no log entries is a run that stopped witnessing.
