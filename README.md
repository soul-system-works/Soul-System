# Soul System

A living philosophy and operational framework for human-AI collaborative work
in research, science, engineering, and coding.

---

## What This Is

Most AI-assisted work fails not because the AI lacks capability
but because the problem was never understood deeply enough before work began.
The AI amplifies whatever framing it receives — including incomplete framing.

This system addresses that. It is a philosophy, a set of operational documents,
and a growing collection of skills and hooks that together create a more
principled, self-correcting approach to AI-assisted work.

It is not a prompt collection. It is not a workflow template.
It is closer to a craft tradition — principles forged from real failure patterns
that practitioners have recognized as true.

## How to Start

**1. Read the Soul**
`philosophy/the-soul.md`
Not as instructions. As a philosophy that must be understood before it can be enacted.
This is the one document that rewards slow reading.

**2. Copy the operations folder into your project**
```
CLAUDE.md
witness-log-format.md
council-synthesis.md
autonomous-session-template.md
```
Record the system version you copied from `SYSTEM-VERSION.md`.

**3. Fill in the problem slot**
Open `autonomous-session-template.md`
Insert your problem. Define your check-in cadence. Start the session.

**4. Read the Witness log**
Not the solution. The log.
The quality of the log tells you more about what happened than the output does.

**5. Contribute what you find**
If your run surfaces a finding or a proposed amendment see `CONTRIBUTING.md`.
The system grows through honest evidence from real use.

---

## What This Is Not

This is not a guarantee of better output.
It is a guarantee of more honest process.

A rich Witness log from a messy run is a successful exercise.
A smooth run with no log entries is a run that stopped witnessing.
