# Governance

This system is governed by its own philosophy.

Changes to this repo follow the amendment process in `council-synthesis.md`.
Evidence from Witness logs must be cited. No amendment is accepted without it.

---

## Roles

**Maintainers** — hold merge authority. Must have Witness log evidence from real use.
Initially the founder. Expanded by invitation.

**Contributors** — anyone with findings or amendments supported by Witness log evidence.
See `CONTRIBUTING.md`.

**Observers** — anyone using the system in their own projects. No obligations.

---

## What Maintainers Can Do

- Accept or return amendments following the amendment process
- Merge accepted amendments into canonical documents
- Update the system version in `SYSTEM-VERSION.md`
- Invite new maintainers
- Add registry entries

## What Maintainers Cannot Do

- Accept amendments without Witness log evidence
- Accept amendments that cannot answer all three amendment questions
- Amend the Soul under urgency or pressure
- Remove any document version from the amendment record
- Act unilaterally on Soul amendments — minimum two maintainers must deliberate

---

## Decision Thresholds

| What | Who | Notes |
|---|---|---|
| Operational documents | One maintainer | Faster cadence |
| The Soul | Two maintainers minimum | Skeptic role must be recorded |
| Governance | All maintainers, unanimous | Rarely changes |

---

## Versioning

The system version is tracked in `SYSTEM-VERSION.md`.
Filenames are stable and never include version numbers.
Git tags mark system version increments.

System version format: `MAJOR.MINOR.PATCH`
- PATCH — skills, hooks, commons entries, registry entries
- MINOR — operational documents amended
- MAJOR — the Soul amended

Prior versions are never discarded. All versions retained in `amendments/accepted/`.

---

## Disputes

Invoke the Skeptic formally. Both positions stated. Skeptic challenges both.
If unresolved — file as a Finding. Revisit when more evidence arrives.
Evidence resolves disputes. Argument rarely does.

---

## Forking

Fork freely. A fork that diverges significantly from the Soul should rename itself.
Improvements are encouraged back through the amendment process.
